## 2. Executive Summary

CasualStep adalah toko online khusus sepatu casual yang membantu pelanggan mencari, membandingkan, dan membeli sepatu berdasarkan ukuran, warna, model, brand, harga, dan kebutuhan aktivitas harian. Produk ini dirancang untuk mengurangi hambatan umum saat membeli sepatu secara online, seperti salah ukuran, informasi stok tidak jelas, foto produk kurang detail, proses checkout panjang, dan status pengiriman tidak transparan.

Website akan menyediakan katalog produk yang mudah digunakan, panduan ukuran, detail varian, ulasan pelanggan, wishlist, promo, checkout aman, serta dashboard admin untuk mengelola produk, stok, pesanan, promo, dan laporan penjualan.

## 3. Problem Statement

Pembeli sepatu casual online sering mengalami beberapa masalah berikut:

- Sulit memastikan ukuran sepatu yang sesuai.
- Informasi stok per ukuran dan warna tidak selalu jelas.
- Foto produk terbatas sehingga bahan, bentuk, dan warna sulit dinilai.
- Filter produk kurang spesifik untuk kebutuhan sepatu.
- Checkout terlalu panjang dan membingungkan.
- Status pembayaran dan pengiriman tidak transparan.
- Proses retur tidak jelas ketika ukuran tidak cocok atau produk cacat.

Dibutuhkan toko online yang fokus pada pengalaman membeli sepatu, dengan informasi produk yang lengkap, alur pembelian sederhana, serta manajemen stok varian yang akurat.

## 4. Product Vision

Menjadi toko online sepatu casual yang praktis, terpercaya, dan nyaman digunakan, dengan pengalaman belanja yang membantu pelanggan menemukan sepatu yang tepat tanpa ragu soal ukuran, stok, harga, dan kualitas produk.

## 5. Goals

### 5.1 Business Goals

Dalam 6 bulan setelah peluncuran, produk ditargetkan untuk:

- Meningkatkan penjualan sepatu casual melalui kanal online.
- Mencapai conversion rate minimal 2%.
- Mencapai checkout completion rate minimal 60%.
- Menurunkan cart abandonment rate di bawah 70%.
- Menjaga return rate akibat salah ukuran di bawah 10%.
- Meningkatkan repeat purchase rate melalui wishlist, promo, dan membership sederhana.

### 5.2 User Goals

Pengguna dapat:

- Menemukan sepatu casual sesuai gaya, ukuran, warna, dan budget.
- Melihat informasi produk secara lengkap.
- Memastikan ukuran dengan panduan size chart.
- Menyimpan produk favorit.
- Membeli produk dengan proses checkout yang mudah.
- Melacak status pesanan.
- Mengajukan retur jika produk tidak sesuai.

### 5.3 Admin Goals

Admin dapat:

- Menambah dan memperbarui produk.
- Mengelola stok berdasarkan ukuran dan warna.
- Memproses pesanan.
- Mengatur promo.
- Memverifikasi pembayaran manual jika diperlukan.
- Melihat laporan penjualan dasar.

## 6. Target User

### 6.1 Primary User

- Usia 18-40 tahun.
- Mahasiswa, pekerja muda, freelancer, dan pengguna aktif internet.
- Membutuhkan sepatu casual untuk kuliah, kerja santai, hangout, jalan-jalan, atau aktivitas harian.
- Sensitif terhadap harga, kenyamanan, ukuran, dan desain.

### 6.2 Secondary User

- Pembeli hadiah.
- Kolektor sepatu casual.
- Pelanggan yang mencari promo.
- Customer service.
- Admin operasional toko.

## 7. User Persona

### Persona 1: Raka, Mahasiswa

- Usia: 21 tahun.
- Kebutuhan: Sepatu casual yang nyaman, murah, dan cocok untuk kuliah.
- Pain points:
  - Takut salah ukuran.
  - Mencari promo atau diskon.
  - Butuh review dari pembeli lain.
- Ekspektasi:
  - Ada size chart.
  - Ada filter harga dan ukuran.
  - Checkout bisa dilakukan cepat dari HP.

- Usia: 27 tahun.
- Kebutuhan: Sepatu casual rapi untuk kerja santai dan hangout.
- Pain points:
  - Tidak punya banyak waktu membandingkan produk.
  - Butuh foto produk yang jelas.
  - Ingin pengiriman cepat.
- Ekspektasi:
  - Filter cepat.
  - Foto detail dari beberapa sudut.
  - Alamat tersimpan dan checkout singkat.

### Persona 3: Budi, Admin Toko

- Usia: 32 tahun.
- Kebutuhan: Mengelola katalog, stok, pesanan, dan promo.
- Pain points:
  - Stok sering berbeda antar ukuran.
  - Sulit memantau pesanan masuk.
  - Laporan penjualan masih manual.
- Ekspektasi:
  - Dashboard ringkas.
  - Stok per varian.

## 11. Functional Requirements

### 11.1 Homepage

Homepage berfungsi sebagai halaman utama untuk memperkenalkan brand, menampilkan kategori produk, promo, dan produk unggulan.

Komponen:

- Header dengan logo, search bar, menu navigasi, login, wishlist, dan cart.
- Hero banner.
- Kategori utama.
- Produk best seller.
- Produk terbaru.
- Promo aktif.
- Keunggulan toko.
- Testimoni pelanggan.
- Footer.

Acceptance criteria:

- Pengguna dapat membuka homepage dari desktop dan mobile.
- Search bar terlihat jelas.
- Banner promo dapat diklik menuju katalog terfilter atau halaman promo.
- Produk unggulan dapat diklik menuju detail produk.
- Cart icon menampilkan jumlah item.

### 11.2 Katalog Produk

Katalog menampilkan daftar sepatu casual yang dapat difilter dan diurutkan.

Informasi pada product card:

- Foto utama.
- Nama produk.
- Brand.
- Harga.
- Harga diskon jika ada.
- Rating.
- Jumlah ulasan.
- Warna tersedia.
- Label promo, baru, best seller, atau stok terbatas.

Filter:

- Kategori sepatu.
- Gender.
- Ukuran.
- Warna.
- Brand.
- Rentang harga.
- Rating.
- Ketersediaan stok.
- Bahan.

Sorting:

- Terbaru.
- Terlaris.
- Harga terendah.
- Harga tertinggi.
- Rating tertinggi.
- Diskon terbesar.

Acceptance criteria:

- Sistem menampilkan produk sesuai filter yang dipilih.
- Pengguna dapat menghapus filter aktif.
- Sorting mengubah urutan produk tanpa menghilangkan filter.
- Produk stok habis diberi label jelas.

### 11.3 Search Produk

Search membantu pengguna menemukan produk berdasarkan keyword.

Search mendukung:

- Nama produk.
- Brand.
- Kategori.
- Warna.
- SKU.
- Keyword seperti “sepatu putih”, “sneakers hitam”, “canvas pria”.

Acceptance criteria:

- Search menampilkan hasil relevan.
- Jika tidak ada hasil, sistem menampilkan empty state dan rekomendasi produk populer.
- Search dapat digunakan di desktop dan mobile.
- Search tidak case-sensitive.

### 11.4 Detail Produk

Halaman detail produk memberikan informasi lengkap agar pembeli yakin sebelum membeli.

Komponen:

- Galeri foto minimal 4 foto.
- Nama produk.
- Brand.
- Harga.
- Diskon.
- Rating dan jumlah ulasan.
- Pilihan warna.
- Pilihan ukuran.
- Stok per varian.
- Deskripsi produk.
- Spesifikasi produk.
- Panduan ukuran.
- Info pengiriman.
- Tombol tambah ke cart.
- Tombol beli sekarang.
- Tombol wishlist.
- Ulasan pelanggan.
- Produk rekomendasi.

Spesifikasi produk:

- Bahan upper.
- Bahan outsole.
- Bahan insole.
- Jenis model.
- Gender.
- Berat produk.
- Tinggi sol.
- Negara produksi.
- Cara perawatan.

Acceptance criteria:

- Pengguna wajib memilih ukuran dan warna sebelum menambahkan ke cart.
- Varian stok habis tidak dapat dibeli.
- Foto produk dapat diperbesar.
- Size chart dapat dibuka dari halaman detail.
- Harga diskon tampil jelas jika promo aktif.

### 11.5 Panduan Ukuran

Panduan ukuran membantu mengurangi retur akibat salah ukuran.

Komponen:

- Tabel ukuran EU, US, UK, dan panjang kaki dalam cm.
- Instruksi mengukur panjang kaki.
- Rekomendasi ukuran berdasarkan panjang kaki.
- Catatan fit produk: normal fit, narrow fit, wide fit.

Acceptance criteria:

- Size chart tersedia di detail produk.
- Admin dapat mengatur catatan fit per produk.
- Pengguna dapat melihat ukuran dalam cm.

### 11.6 Wishlist

Wishlist memungkinkan pengguna menyimpan produk yang disukai.

Acceptance criteria:

- Pengguna login dapat menambahkan produk ke wishlist.
- Produk wishlist dapat dipindahkan ke cart.
- Produk yang stok habis tetap tampil dengan label stok habis.
- Pengguna dapat menghapus produk dari wishlist.

### 11.7 Cart

Cart menyimpan produk sebelum checkout.

Komponen:

- Daftar produk.
- Foto produk.
- Nama produk.
- Ukuran dan warna.
- Harga satuan.
- Kuantitas.
- Subtotal.
- Tombol hapus.
- Input voucher.
- Ringkasan total.

Acceptance criteria:

- Pengguna dapat mengubah kuantitas.
- Sistem mencegah kuantitas melebihi stok.
- Total harga diperbarui otomatis.
- Produk stok habis diberi peringatan sebelum checkout.

### 11.8 Checkout

Checkout adalah proses akhir pembelian.

Tahapan checkout:

1. Login, register, atau checkout sebagai guest.
2. Isi data penerima.
3. Pilih alamat.
4. Pilih kurir.
5. Gunakan voucher.
6. Pilih metode pembayaran.
7. Review pesanan.
8. Konfirmasi pesanan.

Data penerima:

- Nama lengkap.
- Nomor telepon.
- Email.
- Provinsi.
- Kota/kabupaten.
- Kecamatan.
- Kode pos.
- Alamat lengkap.
- Catatan pengiriman.

Acceptance criteria:

- Pengguna dapat checkout sebagai guest jika fitur diaktifkan.
- Sistem menampilkan ringkasan pesanan sebelum pembayaran.
- Ongkir dihitung sebelum pembayaran.
- Voucher valid langsung memotong total.
- Pesanan dibuat setelah pengguna konfirmasi.

### 11.9 Pembayaran

Metode pembayaran:

- Virtual account.
- E-wallet.
- QRIS.
- Kartu debit/kredit.
- Transfer manual untuk MVP jika payment gateway belum tersedia.

Acceptance criteria:

- Sistem menampilkan instruksi pembayaran.
- Status pembayaran otomatis diperbarui jika memakai payment gateway.
- Admin dapat memverifikasi pembayaran manual.
- Pesanan dapat otomatis dibatalkan jika pembayaran melewati batas waktu.

### 11.10 Status dan Riwayat Pesanan

Status pesanan:

- Menunggu pembayaran.
- Pembayaran diverifikasi.
- Diproses.
- Dikemas.
- Dikirim.
- Selesai.
- Dibatalkan.
- Retur diajukan.
- Retur disetujui.
- Retur ditolak.

Acceptance criteria:

- Pengguna dapat melihat riwayat pesanan.
- Detail pesanan menampilkan produk, alamat, ongkir, diskon, total, dan status.
- Nomor resi tampil setelah pesanan dikirim.
- Pengguna dapat memberi ulasan setelah pesanan selesai.

### 11.11 Ulasan Produk

Komponen ulasan:

- Rating 1-5.
- Komentar.
- Foto dari pembeli.
- Ukuran yang dibeli.
- Kesesuaian ukuran: kecil, pas, besar.

Acceptance criteria:

- Hanya pembeli yang sudah menyelesaikan pesanan dapat memberi ulasan.
- Rating rata-rata diperbarui otomatis.
- Admin dapat menyembunyikan ulasan yang melanggar aturan.

### 11.12 Retur dan Refund

Alasan retur:

- Ukuran tidak sesuai.
- Produk cacat.
- Produk salah kirim.
- Warna/model tidak sesuai pesanan.

Aturan retur:

- Retur dapat diajukan maksimal 7 hari setelah barang diterima.
- Produk belum digunakan di luar proses coba ukuran.
- Tag dan kemasan masih tersedia jika diwajibkan.

Alur retur:

1. Pengguna membuka detail pesanan.
2. Pengguna mengajukan retur.
3. Pengguna memilih alasan.
4. Pengguna mengunggah foto bukti.
5. Admin meninjau pengajuan.
6. Admin menyetujui atau menolak.
7. Jika disetujui, pengguna mengirim barang kembali.
8. Admin memproses penggantian ukuran, voucher, atau refund.

Acceptance criteria:

- Retur hanya dapat diajukan untuk pesanan selesai.
- Alasan retur wajib diisi.
- Foto bukti wajib untuk produk cacat/salah kirim.
- Status retur dapat dipantau pengguna.

## 12. Admin Requirements

### 12.1 Dashboard Admin

Komponen:

- Total penjualan hari ini.
- Total pesanan baru.
- Pesanan menunggu pembayaran.
- Pesanan perlu dikirim.
- Produk stok menipis.
- Produk terlaris.
- Grafik penjualan harian/mingguan/bulanan.

Acceptance criteria:

- Admin dapat melihat ringkasan performa toko.
- Data dashboard dapat difilter berdasarkan periode.
- Produk stok menipis tampil berdasarkan threshold.

### 12.2 Manajemen Produk

Admin dapat:

- Menambah produk.
- Mengedit produk.
- Menonaktifkan produk.
- Mengunggah foto produk.
- Mengatur kategori.
- Mengatur brand.
- Mengatur harga.
- Mengatur diskon.
- Mengatur deskripsi.
- Mengatur spesifikasi.
- Mengatur varian ukuran dan warna.

Acceptance criteria:

### 12.3 Manajemen Stok

Admin dapat:

- Menambah stok per varian.
- Mengurangi stok per varian.
- Melihat riwayat perubahan stok.
- Mengatur threshold stok menipis.

Acceptance criteria:

- Setiap kombinasi ukuran dan warna memiliki stok sendiri.
- Sistem mencegah checkout jika stok tidak cukup.

### 12.4 Manajemen Pesanan

Admin dapat:

- Melihat daftar pesanan.
- Memfilter pesanan berdasarkan status.
- Melihat detail pesanan.
- Mengubah status pesanan.

Acceptance criteria:

- Perubahan status tercatat di riwayat pesanan.
- Nomor resi wajib diisi sebelum status menjadi dikirim.

### 12.5 Manajemen Promo

Jenis promo:

- Diskon produk.
- Voucher potongan harga.
- Voucher gratis ongkir.
- Flash sale sederhana.
- Bundling produk.

Aturan promo:

- Periode aktif.
- Minimum pembelian.
- Kuota penggunaan.
- Produk atau kategori tertentu.
- Pengguna baru atau semua pengguna.

Acceptance criteria:

- Promo hanya berlaku dalam periode aktif.
- Voucher tidak dapat digunakan jika syarat tidak terpenuhi.
- Admin dapat melihat jumlah penggunaan voucher.

### 12.6 Laporan Penjualan

Laporan:

- Total omzet.
- Jumlah transaksi.
- Average order value.
- Penjualan per produk.
- Produk terlaris.
- Kategori terlaris.
- Promo paling banyak digunakan.
- Return rate.

Acceptance criteria:

- Admin dapat memfilter laporan berdasarkan tanggal.
- Admin dapat mengekspor laporan ke CSV.
- Total laporan hanya menghitung transaksi berhasil.

## 15. Non-Functional Requirements

### Performance

- Homepage dimuat maksimal 3 detik pada koneksi 4G stabil.
- Katalog dengan 100 produk dimuat maksimal 4 detik.
- Search menampilkan hasil maksimal 1 detik setelah query diproses.
- Gambar produk menggunakan format terkompresi dan lazy loading.

### Security

- Website menggunakan HTTPS.
- Password disimpan dengan hashing aman.
- Validasi input dilakukan di frontend dan backend.
- Proteksi terhadap SQL injection, XSS, CSRF, dan brute force login.
- Role admin dipisahkan dari customer.
- Data pembayaran tidak disimpan langsung jika memakai payment gateway.

### Reliability

- Sistem menjaga konsistensi stok saat checkout.
- Pesanan tidak boleh ganda akibat klik berulang.
- Sistem memiliki mekanisme timeout pembayaran.
- Error penting tercatat dalam log.

### Usability

- Tampilan responsive untuk mobile dan desktop.
- CTA utama mudah ditemukan.
- Filter katalog mudah digunakan.
- Informasi ukuran, stok, harga, dan pengiriman jelas.
- Form checkout tidak meminta data yang tidak diperlukan.

### Accessibility

- Kontras teks cukup jelas.
- Gambar memiliki alt text.
- Form memiliki label.
- Website dapat digunakan dengan keyboard untuk alur utama.
- Error form ditampilkan dengan pesan yang mudah dipahami.

### Scalability

- Struktur data mendukung banyak kategori, produk, dan varian.
- Sistem dapat menangani peningkatan transaksi.
- Media produk dapat dipindahkan ke object storage/CDN.

## 16. Integrasi Pihak Ketiga

## 21. Risiko dan Mitigasi

| Risiko                         | Dampak                      | Mitigasi                                            |
| ------------------------------ | --------------------------- | --------------------------------------------------- |
| Pengguna salah memilih ukuran  | Retur meningkat             | Size chart, catatan fit, review kesesuaian ukuran   |
| Stok tidak akurat              | Pesanan gagal atau komplain | Stok per varian, validasi saat checkout, audit stok |
| Checkout terlalu panjang       | Cart abandonment tinggi     | Checkout ringkas, alamat tersimpan, guest checkout  |
| Pembayaran tidak terverifikasi | Pesanan tertunda            | Payment gateway otomatis atau verifikasi admin      |
| Foto produk tidak meyakinkan   | Conversion rendah           | Standar foto minimal 4 sudut dan zoom bahan         |
| Website lambat                 | Bounce rate tinggi          | Optimasi gambar, caching, lazy loading              |
| Promo disalahgunakan           | Kerugian margin             | Kuota, minimum pembelian, validasi voucher          |
